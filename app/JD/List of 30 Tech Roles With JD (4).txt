1.5.1 Cybersecurity Analyst
* Job Title Cybersecurity Analyst / SOC Analyst (Security Operations Center)
* Professional Summary A vigilant security professional responsible for protecting an organization's digital assets by monitoring networks for malicious activity. Expert in threat detection, incident response, and vulnerability management. Skilled in using SIEM tools to analyze security events and implementing defensive measures to mitigate risks before they escalate into breaches.
* Core Purpose To provide continuous monitoring and rapid response to security threats, ensuring the confidentiality, integrity, and availability of corporate data.
* Key Responsibilities
   * Monitor security information and event management (SIEM) dashboards for real-time alerts
   * Investigate and triage security incidents to determine their scope and impact
   * Perform regular vulnerability scans and coordinate patching efforts with IT teams
   * Analyze network traffic patterns to identify indicators of compromise (IoCs)
   * Implement and manage security tools like firewalls, IDS/IPS, and endpoint protection (EDR)
   * Conduct initial forensic analysis on compromised systems to understand attack vectors
   * Develop and maintain incident response playbooks and escalation procedures
   * Stay updated on the latest cyber threat landscape and "Zero-Day" vulnerabilities
   * Assist in security audits and compliance checks (ISO 27001, SOC2)
   * Provide security awareness training to employees to prevent phishing and social engineering
   * Review system logs and audit trails to ensure policy compliance
   * Collaborate with engineering teams to harden server and network configurations
* Technical Skills
   * Tools & SIEM Splunk, IBM QRadar, Sentinel, Wireshark, Nessus
   * Networking TCP/IP, DNS, VPN, Firewalls, Proxy Servers, Honeypots
   * Security Frameworks NIST, MITRE ATT&CK, ISO 27001, OWASP
   * Operating Systems Linux (Kali/RHEL), Windows Server, macOS
   * Scripting Python, Bash, PowerShell (for automation)
   * Cloud Security AWS Security Hub, Azure Defender, GuardDuty
   * Core Concepts Cryptography, Identity & Access Management (IAM), Threat Hunting
* Qualifications
   * Bachelor’s degree in Cybersecurity, Computer Science, or Information Technology
   * Strong understanding of network protocols and common attack patterns
   * Experience with security monitoring and incident handling
   * Familiarity with security compliance and regulatory standards
* Preferred Qualifications
   * Certifications: CompTIA Security+, CEH, or GCIH
   * Experience in a 24/7 SOC environment
   * Knowledge of digital forensics and malware analysis
* Key Deliverables
   * Daily security monitoring reports and incident logs
   * Documented vulnerability assessment and remediation plans
   * Hardened network and system security configurations
   * Successfully resolved security incidents with minimal data loss

1.5.2 Ethical Hacker
* Job Title Ethical Hacker / Penetration Tester
* Professional Summary An offensive security specialist who proactively identifies weaknesses in an organization's infrastructure by simulating cyberattacks. Expert in web application testing, network exploitation, and social engineering. Skilled in using advanced hacking tools to discover vulnerabilities and providing detailed technical recommendations to engineering teams to fix them.
* Core Purpose To identify and exploit security flaws before malicious actors can, thereby strengthening the organization's overall security posture.
* Key Responsibilities
   * Perform comprehensive penetration tests on web applications, APIs, and network infrastructure
   * Simulate real-world cyberattacks using various techniques (SQLi, XSS, CSRF, Buffer Overflows)
   * Conduct social engineering simulations, including phishing and physical security testing
   * Perform wireless network security assessments and mobile application testing
   * Document findings in detailed reports, including exploit steps and severity ratings
   * Provide actionable remediation guidance and re-test vulnerabilities after fixes are applied
   * Develop custom scripts and tools to automate repetitive testing tasks
   * Research and develop new exploitation techniques for emerging technologies
   * Present security risks to both technical teams and executive leadership
   * Advise on secure network architecture and application design
   * Stay active in the security community (Bug Bounties, CTFs, Research)
   * Maintain strict ethical standards and follow pre-defined "Rules of Engagement"
* Technical Skills
   * Exploitation Tools Burp Suite, Metasploit, Nmap, Kali Linux, Cobalt Strike
   * Web Security OWASP Top 10, Web Sockets, API Security, JWT Exploitation
   * Languages Python, Ruby, JavaScript, C, SQL
   * Network Hacking Packet Injection, Password Cracking, Man-in-the-Middle (MITM)
   * Cloud Hacking Exploiting AWS S3 buckets, IAM misconfigurations, Lambda security
   * Core Concepts Reverse Engineering, Social Engineering, Privilege Escalation
* Qualifications
   * Bachelor’s degree in Computer Science or a related technical field
   * Strong portfolio of bug bounty findings or security research
   * Deep knowledge of how operating systems and web protocols work
   * Exceptional problem-solving skills and an "attacker's mindset"
* Preferred Qualifications
   * Certifications: OSCP (Offensive Security Certified Professional) or GPEN
   * Experience with Red Teaming and adversary simulation
   * Knowledge of hardware hacking or SCADA security
* Key Deliverables
   * Comprehensive Penetration Testing Reports (Technical & Executive summaries)
   * Step-by-step Proof of Concept (PoC) for discovered vulnerabilities
   * Prioritized remediation roadmaps for engineering teams
   * Customized security scripts for automated vulnerability scanning

1.5.3 Application Security Engineer
* Job Title Application Security Engineer / AppSec Engineer
* Professional Summary A specialized security engineer focused on the "Shift Left" approach, ensuring that security is integrated into every stage of the Software Development Lifecycle (SDLC). Expert in secure coding practices, threat modeling, and automated security testing. Works closely with developers to build inherently secure software and manage the security of third-party dependencies.
* Core Purpose To ensure that the application code and architecture are resilient against attacks and free of critical vulnerabilities from the start.
* Key Responsibilities
   * Conduct security design reviews and "Threat Modeling" sessions for new features
   * Perform manual and automated code reviews to identify security flaws (SAST/DAST)
   * Implement and manage Software Composition Analysis (SCA) to track vulnerable libraries
   * Integrate security scanning tools directly into CI/CD pipelines
   * Collaborate with developers to remediate vulnerabilities found in production or during testing
   * Establish "Secure Coding Standards" and provide training to the engineering team
   * Manage the organization's "Bug Bounty" program and triage external reports
   * Design and implement secure authentication and authorization modules
   * Perform security assessments of third-party APIs and integrations
   * Ensure compliance with industry standards like OWASP and ASVS
   * Automate security testing using tools like Semgrep or SonarQube
   * Monitor production applications for security-related errors and anomalies
* Technical Skills
   * Security Testing SAST, DAST, IAST, SCA (Snyk, Checkmarx, Veracode)
   * Development Java, Python, Node.js, Go (Ability to read and fix code)
   * Concepts OWASP Top 10, OAuth2, OpenID Connect, Encryption, Hashing
   * DevOps GitHub Actions, GitLab CI, Docker, Kubernetes Security
   * Frameworks Spring Security, Django Security, Helmet.js
   * Tools Burp Suite (Basic), Semgrep, OWASP ZAP, Git
* Qualifications
   * Bachelor’s degree in Computer Science or Software Engineering
   * Background in software development with a passion for security
   * Strong understanding of web application vulnerabilities and their fixes
   * Experience with CI/CD and modern development workflows
* Preferred Qualifications
   * Certifications: CASE (Certified Application Security Engineer) or CSSLP
   * Experience with container security and serverless security
   * Knowledge of DevSecOps practices and cultural implementation
* Key Deliverables
   * Threat Model documents for major application components
   * Automated security guardrails within the CI/CD pipeline
   * Standardized secure-coding libraries and guidelines
   * Reduced "Vulnerability Debt" through continuous code auditing
